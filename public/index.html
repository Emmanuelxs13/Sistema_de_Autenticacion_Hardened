¿Qué es Hardened Auth?

Es una base de autenticación segura para aplicaciones Node.js, orientada a reducir fraude, takeover de cuentas y riesgo de sesión.

Servicios de seguridad incluidos

  • Hash de contraseñas con Argon2id (memory-hard).
  • Sesión con JWT en cookies `HttpOnly`, `Secure`, `SameSite`.
  • MFA/2FA TOTP con QR para Google Authenticator.
  • Rate limiting en rutas críticas de autenticación.
  • Anti brute-force con bloqueo temporal por cuenta.
  • Refresh token con rotación para continuidad de sesión.
  • Audit log de eventos de seguridad para trazabilidad.

Qué puedes hacer con la app

  • Registrar usuarios con validación fuerte de contraseña.
  • Configurar y activar MFA en dos pasos.
  • Autenticar con flujo password + TOTP.
  • Renovar sesión con endpoint de refresh.
  • Inspeccionar actividad con endpoint de auditoría.

Endpoints principales

  • POST /api/auth/register
  • POST /api/auth/mfa/setup
  • POST /api/auth/mfa/verify-setup
  • POST /api/auth/login
  • POST /api/auth/login/2fa
  • POST /api/auth/refresh
  • GET /api/auth/audit-log
  • GET /api/auth/me
  • POST /api/auth/logout

1) Registro

Crea un usuario con contraseña hasheada usando Argon2id.

2) Setup MFA

Genera QR y secreto para enrolar Authenticator.

3) Activar MFA

Valida un código TOTP de 6 dígitos para activar segundo factor.

4) Login paso 1

Valida credenciales y obtiene token temporal.

5) Login paso 2 (MFA)

Con token temporal + TOTP se emite JWT en cookie segura.

Sesión protegida

Consulta perfil con cookie HttpOnly y luego cierra sesión.
